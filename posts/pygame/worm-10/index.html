Our First Pygame - Fix wrap, bite me - part 10

Date: 23.01.2024 - Author: Richard Porteous

Fixing screen wrap

To make the screen work I needed to sit down and sletch and talk through my logic. I needed to keep track of the movement in the tail in the same way I kept track in the head.

I placed these in gameobject before the init (mostly last_start_move_pos,last_direction,last_boundary_check), some of the rest are from the player.

    start_move_pos = (0,0)
    end_move_pos = (0,0)
    direction = (0,0)
    min = (0,0)
    max = (0,0)
    boundary_check = "none"
    last_end_move_pos = (0,0)
    last_start_move_pos = (0,0)
    last_direction = (0,0)
    last_boundary_check = "none"

and we change setup next move to look like this (same fields again)

    def setup_next_move(self, direction, end_move_pos):
        self.last_end_move_pos = self.end_move_pos
        self.last_start_move_pos = self.start_move_pos
        self.last_direction = self.direction
        self.last_boundary_check = self.boundary_check

        self.direction = self.fix_direction(direction)
        self.rect.center = end_move_pos
        self.start_move_pos = end_move_pos
        self.end_move_pos = (self.start_move_pos[0] + self.direction[0] * self.tilesize[0], self.start_move_pos[1] + self.direction[1] * self.tilesize[1])

then in move in the end of move check we move/add this to just before the return

self.set_boundary_check("none")

and in complete move we change the function to the following

    def complete_move(self):
        self.last_end_move_pos = self.end_move_pos
        self.last_start_move_pos = self.start_move_pos
        self.last_direction = self.direction
        self.last_boundary_check = self.boundary_check
        
        self.rect.center = self.object_to_follow.last_start_move_pos
        self.start_move_pos = self.object_to_follow.last_start_move_pos
        self.end_move_pos = self.object_to_follow.last_end_move_pos
        self.boundary_check = self.object_to_follow.last_boundary_check
           
        self.direction = self.object_to_follow.last_direction

in the tail move we remove

self.check_boundaries()

and in the tail update at the end we add

self.draw_alter_ego(self.rect,self.image,screen)

I think thats it. So we made some fields available to the tail that the player was using and the follow is slightly simpler than its equivalent in the player head.

The name “draw alto ego” is just me trying to be funny. I rename it later.

Ok now we want the head to bite the tail if it collides and end the game.

So we add this in the gameobject init

        self.collide_rect = Rect(self.rect[0] + self.rect[2]/2, self.rect[1] + self.rect[3]/2, self.rect[2]/2, self.rect[3]/2) #left,top,width,height
        self.just_created = True

we add this in the player

    def collide(self, tails):
        for t in tails:
            if pygame.Rect.colliderect(self.collide_rect, t.collide_rect) and t.just_created == False:
                return True
        return False

we set the fields in player for every move

self.collide_rect.center = self.rect.center

and see if we are colliding at the end of the game loop

    if player.collide(player_tail):
        game_running = False
