Starting our PyGame game

All Pygame games share some code in common. Setup, game-loop, user-input, audio etc. etc. We will start building our game and then reuse code as we go. We will refactor this code library into something reusable as we go.

I’m using VSCode with the python and the Git Graph plugin. (pylance is included with the python plugin). It makes editing the code simpler to use an IDE, though using a texteditor is perfectly fine.

Assuming we are new to games or pygame or python we will begin simply by creating an object that moves and interacts with the environment.

I assume you know some python. However if you just read the python as English it will understandable. for now # is a comment for the rest of the line. Python relies on indentation to group code.

Type the following into myapp.py NB. if you copy paste, paste as plain text to remove the extra wierd html characters

# the pygame imports to make them usable in our game
import pygame
from pygame.locals import *


# Initialize the pygame code
pygame.init()


#​
#
#


# quit the pygame window
pygame.quit()
#the app exits

This will run but do nothing except say something like : pygame 2.5.2 (SDL 2.28.3, Python 3.12.0) Hello from the pygame community. https://www.pygame.org/contribute.html

Ok lets create a window. Replace the 3 comments ‘###’ in the middle with the following code

# set window size
size = width, height = (800, 600)
screen = pygame.display.set_mode(size)
# update the display to see what we set
pygame.display.update()

# control variable
game_running = True


# GAME LOOP
while game_running:
    # event listeners 
    # i.e. listen to key-presses or windows closing etc.
    for event in pygame.event.get():
        if event.type == QUIT:
            # collapse the app
            game_running = False


    pygame.display.update()

Now you should see a window that you can exit.

  • The Quit looks for the event to quit.
  • The game loop makes it keep checking for an event and updating the screen with the current info.
  • The display update draws the window.
  • We set the window size as 800 by 600.

Screen redrawing

Screen redrawing is a very established method of removing images redrawn elsewhere on the screen by simply redrawing everything each time you loop around in the game loop.

Sprites were introduced many many years ago to efficiently redraw only the small bit under the sprite and any small bits over the sprite. This is more efficient for a few objects, but not with many. Pygame does support sprites. I plan to take a quick look at them in a later post, but we will stick with redrawing the whole screen.

PROBLEM: If you drew something to the screen now and then tried to move it, the item would appear in every place it redrew. Great for a paint program but useless for us.
