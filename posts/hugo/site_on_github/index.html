Hugo site on github

Date: 04.01.2024 - Author: Richard Porteous

Steps we follow

  • create 2 github repositories. name.github.io and hugoblog
  • Get Hugo and git
  • Create the hugo site locally
  • init git for hugoblog and add public as a git submodule
  • Add a theme (possibly as a submodule)
  • go to the example site folder and copy the files
  • if necessary, rename config.toml to hugo.toml
  • if the posts use the yaml (—) format not toml add a posts archetype
  • can create your first post
  • ## Run hugo server and take a look
  • check existing content
  • remove or replace default content
  • Test and fix site
  • Publish

Create the github repositories

Go to github.com and if you dont have an account create a free one (for now).

select the repositories tab and create two new repositories. 
hugoblog and another with the account name and ending with .github.io

create hugoblog as private (for now) and the github.io page(s) as public.
This will result in two empty repositories.

Get Hugo and git

The Hugo site has a getting started page which will help you install hugo extended and git. It is also possible you may need to install go.

Create Hugo site locally

I used CMD on windows despite the warning not to do so. I didn’t try anything fancy and it seemed to work fine. I suggest you follow the recommendations.

Make the folder you want to keep your Hugo site repositories in.

Do you see the two commands there. Change you current directory/folder to the one you want to create your site in.

The second is

hugo new site name-of-site

Change directory to that site name and list the files

content holds the pages and posts which are markdown files

static holds the images

themes will soon hold your theme

public - not created yet will be the folder we want to publish to the github.io pages (after running hugo - by itself with no arguments)

hugo.toml is our configuration file. we will adjust this soon

init git for hugoblog and add public as a git submodule

in the local blog site folder mine is gameexperiments (see above).

replace the obvious like account-name with your account name etc. You will find these instructions in the empty hugoblog github repository you created

echo "# hugoblog" >> README.md
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/account-name/hugoblog.git
git push -u origin main

Add a theme

Hugo has an example using a theme called Ananke. I like that theme because it has so much information on configuration.

I want to note - NOT ALL THEMES ARE EQUAL. Some require extra work and downloads. Please note the themes’ readme.

Head over to Hugo Themes and select a theme you like. My needs are well defined so I chose one that would match my needs. I chose Tikva.

Tikva’s instructions are simple.

cd themes
git clone https://github.com/geschke/hugo-tikva.git

cd ..
git submodule add -f https://github.com/geschke/hugo-tikva themes/hugo-tikva

next we need to run this command to set our theme.

hugo -t "hugo-tikva"

go to the example site folder and copy the files

Copy the example, usually this is the content, layouts, and static folder to the top-level folder (my-themed-site) overwriting the empty folders there.

rename config.toml to hugo.toml

copy the config.toml to the same folder, remove the default hugo.toml. Then rename config.toml to hugo.toml or use

cp themes\hugo-tikva\exampleSite\config.toml hugo.toml

Notice the header_background.jpg and the folder structure. I found that this works with some minor changes to our hugo.toml file

change the 
1. baseurl
2. title
3. paginate = 10
4. [params.meta]
   description = "Various coding tutorials"
   keywords = "blog, site, tutorials, coding, software, programming"
   author = "Your Name"
   #"DC.Copyright" = "Your Name"
5. [params.theme]

    bootstrapTheme = 'signa' #use one of the examples here
6. and in [params.theme.header] change image = 'img/header_background.jpg'
7. check the theme readme for additional notes

if the posts use the yaml (—) format not toml add a posts archetype

When creating new posts the top meta of the posts should be the same i.e. yaml or toml etc. I found adding a posts.md in archetypes as explained in archetypesto be the best.

---
date: '{{ .Date }}'
title: '{{ replace .File.ContentBaseName `-` ` ` | title }}'
tags: ["Text","Blog","Post"]
author: "Your Name"
---

for more about the different formats check out https://gohugo.io/content-management/front-matter/#front-matter-formats

can create your first post

hugo new post/mypost.md

edit your new post with some markdown content.

NOTE - have at least 3 non-draft posts before panicking. I found them to not always show up - very annoying. Once you have 3 posts even the drafts = true starts to work without removing existing posts.

Run hugo server and take a look

Its a good time to check the results.

hugo server

If there is an error please check what it is referring to.

open the browser to the address http://localhost:1313/ which is the server you just started.

check any existing content for image links etc. Check the paths are correct

Remove the /Static/ in front of image links in your posts if you had them in your editor. The site removes the need for that so you need to find a plugin in for your editor that respects the hugo paths.

Remove default content

Check all your content and read before deleting. html can be used for posts instead of markdown BUT can also be used for headless etc. so be careful.

And finally we need to remove the additional content from the about, feedback, extra posts, and sections.

Replace existing images with images you want to use.

Test and fix site content

For this theme lets replace a few items.

In the static folder we want to overide the themes icons and header image.

in static place your own

/static/favicon.ico

add an images folder add your top image
 
/static/img/header_background.jpg

the name is determined in the template.

there are also two files favicon-32x32.jpg and favicon-16x16.jpg which could be placed in that folder for devices viewing the site.

Some themes has a hardcoded nav-bar. This one is dynamic.

Publish

to publish

hugo

cd public

git add .

git commit -m "commit message you understand"

git push -u origin main

The last one only works if you have set your online reposity upstream and you have added a git user email and name

NOTEs

images in hugo posts should not include the /static in the path. this can be confusing if you use an ide that doesn’t check for this i.e. VSCode

![image](/img/image.png)

not

![image](/static/img/image.png)

in you main site folder (in my case that would be “gameexperiments”) type in the following

hugo
hugo server

hugo, builds the site hugo server, allows us to see the site locally in a browser

in the browser navigate to the suggested site

http://localhost:1313/

if you dont see your post - remove the line “draft: true”

then to quit the hugo server you could CTRL-C CTRL-C

run hugo again to publish your files. If satisfied.

hugo

now we should init git in the public folder and set it upstream to our github pages site …github.io. once done push the pulic git to origin. (once again REPLACE my account and repository name with yours)

cd public

echo "# richard-porteous.github.io" >> README.md
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/your-account/your-account.github.io.git
git push -u origin main

set as a sub module git submodule add -b main -f https://github.com/your-account/your-account.github.io public

then a second commit and push for all the files

add .
git commit -m "add site files"
git push origin main

after a minute or so open your your-account.github.io site in the browser.

OK now lets add all that code to the hugoblog repository

cd ..

this should place us back in the site code folder

and the add files and commit and push

add .
git commit -m "add initial site files, minus sub modules"
git push origin main

there are a few ways to automate things but for now we add posts, build the site (run hugo). We check by running hugo serve and viewing in browser. Then in the public folder and site folder (check git status) and add files (git add .), commit to the main branch and push to origin.

If your account-name.github.io site does not get the themes or some other change verify the above then check the pages tab for your site in your repository settings.

if still nothing do the same git add/commit/push command on the hugoblog site

different theme - menus not dynamic??

If you chose a theme without a dynamic nav bar (i.e. silhouette-hugo), then to fix the nav-bar in this theme we actually need to edit one of the themes files. The names of folders and files may differ.

navigate to

themes\silhouette-hugo\layouts\partials

edit

top-nav-text-links.html

replace this

<li class="nav-item">
    <a class="nav-link active" href="{{ "categories/development/" | absURL }}">Development</a>
</li>
<li class="nav-item">
    <a class="nav-link active" href="{{ "categories/golang/" | absURL }}">Golang</a>
</li>
<li class="nav-item">
    <a class="nav-link active" href="{{ "about/" | absURL }}">About</a>
</li>

with

<li class="nav-item">
    <a class="nav-link active" href="{{ "post/" | absURL }}">All posts</a>
</li>
<li class="nav-item">
    <a class="nav-link active" href="{{ "about/" | absURL }}">About</a>
</li>

A quick note about compatability when placing an image from the static folder into a post,

1. please remove the word static from the internal link
2. keep the name lowercase and match the case in the markdown
3. tikva uses the img folder for site images not "images"

i.e. ![](/img/blog/site-from-theme-example/theme-example-folder.png)
