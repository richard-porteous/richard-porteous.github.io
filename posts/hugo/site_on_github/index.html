Hugo site on github

Date: 04.01.2024 - Author: Richard Porteous

Steps we follow

1. create 2 github repositories. name.github.io and hugoblog
2. Get Hugo and git
3. Create the hugo site locally
4. init git for hugoblog and add public as a git submodule
5. Add a theme (possibly as a submodule)
6. Test and fix site or theme
7. Add a post and about
10. Publish

Create the github repositories

Go to github.com and if you dont have an account cretae a free one (for now).

select the repositories tab and create two new repositories. 
hugoblog and another with the account name and ending with .github.io

create hugoblog as private (for now) and the github.io page(s) as public.
This will result in two empty repositories.

Get Hugo and git

The Hugo site has a getting started page which will help you install hugo extended and git. It is also possible you may need to install go.

Create Hugo site locally

I used CMD on windows despite the warning not to do so. I didn’t try anything fancy and it seemed to work fine. I suggest you follow the recommendations.

Make the folder you want to keep your Hugo site repositories in.

Do you see the two commands there. Change you current directory/folder to the one you want to create your site in.

The second is

hugo new site name-of-site

Change directory to that site name and list the files

content holds the pages and posts which are markdown files

static holds the images

themes will soon hold your theme

public - not created yet will be the folder we want to publish to the github.io pages

hugo.toml is our configuration file. we will adjust this soon

init git for hugoblog and add public as a git submodule

in the local blog site folder mine is gameexperiments (see above).

replace the obvious like account-name with your account name etc. You will find these instructions in the empty hugoblog github repository you created

echo "# hugoblog" >> README.md
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/account-name/hugoblog.git
git push -u origin main

Add a theme

Hugo has an example on thier site using a theme called Ananke. I like that theme because it has so much information on configuration.

I want to note - NOT ALL THEMES ARE EQUAL. Some require extra work and downloads. Please note the themes’ readme.

Head over to Hugo Themes and select a theme you like. My needs are well defined so I chose one that would match my needs. I chose silhouette-hugo.

Silhouette hugo’s instructions are simple.

cd themes

git clone https://github.com/mattbutton/silhouette-hugo.git

cd ..

git submodule add -f https://github.com/mattbutton/silhouette-hugo themes/hugo-tikva

next we should configure hugo.toml

the default values for hugo.toml are:

baseURL = 'https://example.org/'
languageCode = 'en-us'
title = 'My New Hugo Site'

We will change baseURL, title, and add themes.

baseURL = 'https://richard-porteous.github.io/'
languageCode = 'en-us'
title = 'Game Experiments Blog'
theme = 'silhouette-hugo'

next we need to run this command to set our theme.

hugo -t "silhouette-hugo"

Test and fix site or theme

For this theme lets replace a few items. In the static folder we want to overide the themes icons and header image.

in static place your own

/static/favicon.ico

add an images folder add your top image
 
/static/img/hero-main.jpg

the name is determined in the template.

there are also two files favicon-32x32.jpg and favicon-16x16.jpg which should be placed in that folder for devices viewing the site.

This theme has a hardcoded nav-bar. In content we want:

about.md
a post folder
and in post you can create your first post i.e.

hugo new post/mypost.md

edit your new post with some markdown content.

To fix nav-bar in this theme we actually need to edit one of the themes files.

navigate to

themes\silhouette-hugo\layouts\partials

edit

top-nav-text-links.html

replace this

<li class="nav-item">
    <a class="nav-link active" href="{{ "categories/development/" | absURL }}">Development</a>
</li>
<li class="nav-item">
    <a class="nav-link active" href="{{ "categories/golang/" | absURL }}">Golang</a>
</li>
<li class="nav-item">
    <a class="nav-link active" href="{{ "about/" | absURL }}">About</a>
</li>

with

<li class="nav-item">
    <a class="nav-link active" href="{{ "post/" | absURL }}">All posts</a>
</li>
<li class="nav-item">
    <a class="nav-link active" href="{{ "about/" | absURL }}">About</a>
</li>

NOTE

images in hugo posts should not include the /static in the path ie
![image](/img/image.png)
not
![image](/static/img/image.png)

in you main site folder (in my case that would be “gameexperiments”) type in the following

hugo
hugo server

in the browser navigate to the suggested site

http://localhost:1313/

if you dont see your post - remove the line “draft: true”

then to quit the hugo server you could CTRL-C CTRL-C

run hugo again to publish your files. If satisfied.

hugo

now we should init git in the public folder and set it upstream to our github pages site …github.io. once done push the pulic git to origin. (once again REPLACE my account and repository name with yours)

cd public

echo "# richard-porteous.github.io" >> README.md
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/richard-porteous/richard-porteous.github.io.git
git push -u origin main

set as a sub module git submodule add -b main -f https://github.com/richard-porteous/richard-porteous.github.io public

then

add .
git commit -m "add site files"
git push origin main

after a minute or so open your account-name.github.io site in the browser.

OK now lets add all that code to the hugoblog repository

cd ..

this should place us back in the site code folder

and the add files and commit and push

add .
git commit -m "add initial site files, minus sub modules"
git push origin main

there are a few ways to automate things but for now we add posts, build the site (run hugo). We check by running hugo serve and viewing in browser. Then in the public folder and site folder (check git status) and add files (git add .), commit to the main branch and push to origin.

If your account-name.github.io site does not get the themes or some other change verify the above then check the pages tab for your site in your repository settings.

if still nothing do the same git add/commit/push command on the hugoblog site

if still having problems check with hugo. it may be something simple that was overlooked, or there is a missing step here. If you find an easy way to solve it drop me a note. thanks.
