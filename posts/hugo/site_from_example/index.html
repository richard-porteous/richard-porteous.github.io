Use a Hugo theme example

Date: 04.01.2024 - Author: Richard Porteous

Using the Tikva theme example site

If you want to start a site and have a definite theme in mind. If its not 100% compatible with what you have been doing, I find this method to be best.

  1. create your hugo site
  2. grab a theme from the theme store and clone it into themes
  3. go to its example site folder and copy the site into yours
  4. if an older theme, rename config.toml to hugo.toml
  5. if the posts use the yaml (—) format not toml add a posts archetype
  6. check any existing content for internal links. Check the paths are correct
  7. run the hugo server

Replace the static files with files you want, set your header background image if you want one. If mutliple themes choose one.

If post headers are different take care to use what they used, unless you know how to fix that. My first few attempts my hugo generated post headers failed. But in the end I have what I want so I am happy with the results. Ok let’s get started!

Create your Hugo site

On your local device/pc change to the directory you want the site to keep the site code in. This could be the same as what you used in the last tutorial. In my case it is blog_repos and I want to call the site folder my-themed-site.

Then create the hugo site from the terminal command line.

hugo new site my-themed-site

Change directory to the directory my-themed-site and list the files.

grab a theme from the theme store and clone it into themes

Next change to the themes folder. We will clone our theme here.

Head over to All Hugo Themes. If you are following this step by step, then grab the Hugo Tikva them.

Take a look at the readme (usually displayed below the repository) - they are an excellent source of info.

cd themes
git clone https://github.com/geschke/hugo-tikva.git

Note: The next instruction is to optionally create a submodule, we can’t do that yet as we haven’t initialized a git repository in our local top-level folder. Also I may refer to this as site folder or main, sorry old habits die hard. When I do refer to the one on github I will specifically say it is on github and not local.

Ok back up to my-themed-site folder and type in the following.

echo "# build from a tikva example site" >> README.md

git init
git add .
git commit -m "first commit"
git branch -M main

You will see a warning about the git from the theme, its ok it wont include it, its optional and as long as you can access the original repository you are ok, but consider setting it as a submodule later.

go to its example site folder and copy the site into yours

Copy the example, usually this is the content, layouts, and static folder to the top-level folder (my-themed-site) overwriting the empty folders there.

if an older theme, rename config.toml to hugo.toml

copy the config.toml to the same folder, remove the default hugo.toml. Then rename config.toml to hugo.toml

Notice the header_background.jpg and the folder structure. I found that this works with some minor changes to our hugo.toml file

change the 
1. baseurl
2. title
3. paginate = 10
4. [params.meta]
   description = "Various coding tutorials"
   keywords = "blog, site, tutorials, coding, software, programming"
   author = "Your Name"
   #"DC.Copyright" = "Your Name"
5. [params.theme]

    bootstrapTheme = 'signa' #use one of the examples here
6. and in [params.theme.header] change image = 'img/header_background.jpg'
7. check the theme readme for additional notes

And finally we need to remove the additional content from the about, feedback, extra posts, and sections.

Test the site again - missing posts? check the post headers again!

Make sure you have a least 3 non-draft posts.
I noticed less than three non draft posts failed to show up

When creating new posts the top meta of the posts should be the same i.e. yaml or toml etc. I found adding a posts.md in archetypes as explained in archetypesto be the best.

---
date: '{{ .Date }}'
title: '{{ replace .File.ContentBaseName `-` ` ` | title }}'
tags: ["Text","Blog","Post"]
author: "Your Name"
---

Finish testing and create another git commit. It is likely time to set your “theme” and “public” folder as git submodules and set the git remote(s). refer to previous post

Enjoy!

NOTES

for more about the different formats check out https://gohugo.io/content-management/front-matter/#front-matter-formats

Make sure you have a least 3 non-draft posts before trying to figure if there is a problem. Seems there is a bug, just not yours.

A quick note about compatability when placing an image from the static folder into a post,

1. please remove the word static from the internal link
2. keep the name lowercase and match the case in the markdown
3. tikva uses the img folder for site images not "images"

i.e. ![](/img/blog/site-from-theme-example/theme-example-folder.png)
