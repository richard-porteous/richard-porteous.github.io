Using Git

Date: 30.01.2023 - Author: Richard Porteous

It is highly recommended that you use GIT so you keep track of your chnges and don’t lose your work.

What we will do this Lesson

  • Learn to add your project to a git repository.
  • Know how to use a free online host.

git-gui, Sourcetree, etc

Git is simple to learn. I suggest using a Git Client for viewing your history or everything else if you are a beginner.

We use the command line wherever we can.

I use GitHub as my remote host, but choose whichever you tink is best. Some activity we do relies on GitHub pages so you may want to consider them.

.gitignore

This file removes temporary files from your commits. It simply tells git what it can safely ignore.

Godot can provide the .gitignore for you when you create the project.

Create the repository

you can do this with or without an online host.

I usually will create the host folder first then create the local repository and then link them. But you do not need the host for these projects we will do. For your projects, you should create an online account.

Get a GitHub account

If you are younger than 13 you will need permission.

  • Go to https://github.com/ and create an account if you don’t already have one.

  • You may use/choose a name other than the default master for the main branch. Common choices are main or develop. Remember your choice. Once you have created your profile, you can do this from your profile icon top right. Select Settings. On the Settings page choose Repositories (on the left under code planning and automation). The top choice should be a way to change your default branch name.

  • create an empty repository using the NEW REPOSITORY option. add the name HeroRPG (whatever) and keep it private (unless you plan to share).

Accept the changes

Look near the bottom of the page you will get the same instructions with the correct URL from when you created the GitHub repository. you can simply use these to continue.

(URL should look like https://github.com/YOUR_NAME/YOUR_PROJECT.git … where YOUR_NAME is the name you chose during signup, and YOUR_PROJECT is the name of your project)

NEW

(if not empty zip it up just in case and keep that zip handy)

with git installed
Open a command prompt window (cmd or your computers terminal app)
go to your project folder (top level)

git init
git remote add origin https://github.com/YOUR_NAME/YOUR_PROJECT.git
git pull
git add .
git checkout main
git status

note main and master branch names depend on that used in your github repository.

How do we save all future changes

When you start working on your project again you need to do regular commits (saves). Usually, every time you work on a new thing or more than a few hours of work you could save the changes in a commit.

Stage all changed files: (without the quotes)

  • “git add .”

this adds all the changes to a staging area - only the staged changes are saved.

Make sure all is good:0

  • git status

Once the status is checked we commit the changes

  • git commit -m “Message that describes what this change does”

Or to fix the commit message

  • git commit –amend -m “Put your corrected message here”

And finally, we want to push the repository (Sync) so all our changes are safe in our online host (GitHub etc)

  • git push

This will keep your work safe if you do it at the end of each session - but may not be neat. I highly recommend any git client app. remember to (stage, commit, and push) … git Clients will often show the list of changed files and all you need to do is push a button for staging, for committing, and to finally push the changes. or sticking with command line - here are two starting points. https://github.com/git-guides https://git-scm.com/docs

Built in

You can use git directly from a lot of tools and ide’s these days. if you can i suggest you do that or the command line.

VSCode which I suggest you use in many tutorials has Git Support with a few plugins which give you extra features. I like git graph simply because I like to visualize the structure of my git repository. It adds almost everything you need, you just need to explore it a bit.

Git client

These are really useful when you have more than one source repository and are trying to more than the ordinary (command line is still better). There focus is to help you manage your git repositories, use branches, and many other things almost all visually. Helps reduce the need to understand and memorise yet another tool.

Of the Clients I currently recommend GitAhead. Followed by source-tree. There are many others, I suggest you try a few. I have had one or two git client apps really do a lot of damage but I was trying something stupid anyway.

Git Gui comes with git, not as intuitive but you dont have anything extra to install.

On the cmd line type git-gui or search with the search bar to find the app ~ you’ll have to select the new “repository” the first time, it will remember for the second time.

In the app select “Repository”, then select “visualize all branch history”
