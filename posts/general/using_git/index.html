Using Git

Date: 30.01.2023 - Author: Richard Porteous

Get Git

It is highly recommended that you use GIT to keep track of your changes and prevent loss of your work.

I usuall head to https://git-scm.com/downloads If you have linux you likely have git already, but doesn’t hurt to check out the above.

Some Concepts

From the horses mouth

Its always better to look at official documentation - git is simple you should start here https://git-scm.com/doc

where are my file (versions)

git creates a repository which is the hidden folder called .git that git likes to keep a full repository for you on your local computer.

the command git initi is usually used for this.

Do you need an online account

No. But often its the best way to keep your git repository safe from deletion. (well you could still delete it online but you have to be intentional)

git file .gitignore

This file removes unwanted files from your commits. It simply tells git what it can safely ignore.

Some software and github.com provides a preconfigured .gitignore for you when you create a project or repository.

commands you may use frequently

  • add - this ‘stages’ files so they are ready to commit
  • commit - this adds an entry to a branch. each commit has a short message to identify its purpose.
  • branch - this usually consists of a number of commits in cronological order and can be given a name
  • push - you use this to ‘push’ your commits (branch) to an upstream repository. Often the upstream repository is one you have on a host or another computer.
  • pull - this is used mostly when you want the branch on your repository to contain other changes made on the host to the host. Push & pull are used synchronize your repository with the host.
  • merge - the common way to get changes on one branch to be placed on the main branch. Yes you can have many branches.

GUI

to create a repository

The remote command wont work. The URL will need to match the one for the ‘Host’. i.e. if you created a empty repository on github it will give you a URL to use.

GitHub instructions https://docs.github.com/en/get-started/importing-your-projects-to-github/importing-source-code-to-github/adding-locally-hosted-code-to-github

echo "# Readme" >> README.md
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/account-name/repository-name.git
git push -u origin main

How do we save all future changes

When you start working on your project again you need to do regular commits (saves). Usually, every time you work on a new thing or more than a few hours of work you could save the changes in a commit.

Stage all changed files: (without the quotes)

  • “git add .”

this adds all the changes to a staging area - only the staged changes are saved.

Make sure all is good:0

  • git status

Once the status is checked we commit the changes

  • git commit -m “Message that describes what this change does”

And finally, we want to push the repository (Sync) so all our changes are safe in our online host (GitHub etc)

  • git push

This is very basic info. It will keep your work safe if you do it at the end of each session - but may not be neat. I highly recommend any git client app. remember to stage (add), commit, and push … git Clients will often show the list of changed files and all you need to do is push a button for staging, for committing, and to finally push the changes. or sticking with command line - here are two starting points. https://github.com/git-guides https://git-scm.com/docs

Built in

You can use git directly from a lot of tools and ide’s these days. if you can i suggest you do that or the command line.

VSCode which I suggest you use in many tutorials has Git Support with a few plugins which give you extra features. I like git graph simply because I like to visualize the structure of my git repository. It adds almost everything you need, you just need to explore it a bit.
