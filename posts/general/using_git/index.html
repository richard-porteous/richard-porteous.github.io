Using Git

Date: 30.01.2023 - Author: Richard Porteous

It is highly recommended that you use GIT to keep track of your changes and prevent loss of your work.

What we will do this Lesson

  • Learn to add your project to a git repository.
  • Know how to use a free online host.

git-gui, Sourcetree, etc

Git is simple to learn. We use the terminal here, though a Git Client (UI) can help visualize better.

I use GitHub as my remote host, but choose whichever you think is best. Some activity we do relies on GitHub pages so you may want to consider GitHub.

.gitignore

This file removes temporary files from your commits. It simply tells git what it can safely ignore.

Some software provides a preconfigured .gitignore for you when you create a project.

I usually will create the host repository first then create the local repository. But you do not need the host for most of these projects. For your own projects, you should create an online free account and ad the repository following next two sections

Get a GitHub account

If you are younger than 13 you will need permission from parent or guardian.

  • Go to https://github.com/ and create an account if you don’t already have one.

Create the repository

Once you have created your account profile

Optional

  • you can do rename the default branch from your profile icon top right. Select Settings. On the Settings page choose Repositories (on the left under code planning and automation). The top choice should be a way to change your default branch name.

Required

  • create an empty repository using the NEW REPOSITORY option. You will name it too depending on your needs i.e. your-username.github.io Keep it private unless you plan to share (see usage agreement) i.e. your-username.github.io should be public.

Accept the changes

Look near the bottom of the page you will get the instructions with the correct URL to create the local repository.

URL should look like 
https://github.com/YOUR_NAME/YOUR_PROJECT.git
YOUR_NAME is the name you chose during signup
YOUR_PROJECT is the name of your project

with git installed Open a command prompt window (cmd or your computers terminal app) go to your new empty project folder (top level)

If you created content online (i.e. README, .gitignore etc.)

This makes some assumptions

git init
git remote add origin https://github.com/YOUR_NAME/YOUR_PROJECT.git
git pull
git add .
git branch -M main
git status

If you did not create content online

echo "# Readme" >> README.md
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/account-name/repository-name.git
git push -u origin main

How do we save all future changes

When you start working on your project again you need to do regular commits (saves). Usually, every time you work on a new thing or more than a few hours of work you could save the changes in a commit.

Stage all changed files: (without the quotes)

  • “git add .”

this adds all the changes to a staging area - only the staged changes are saved.

Make sure all is good:0

  • git status

Once the status is checked we commit the changes

  • git commit -m “Message that describes what this change does”

And finally, we want to push the repository (Sync) so all our changes are safe in our online host (GitHub etc)

  • git push

This is very basic info. It will keep your work safe if you do it at the end of each session - but may not be neat. I highly recommend any git client app. remember to stage (add), commit, and push … git Clients will often show the list of changed files and all you need to do is push a button for staging, for committing, and to finally push the changes. or sticking with command line - here are two starting points. https://github.com/git-guides https://git-scm.com/docs

Built in

You can use git directly from a lot of tools and ide’s these days. if you can i suggest you do that or the command line.

VSCode which I suggest you use in many tutorials has Git Support with a few plugins which give you extra features. I like git graph simply because I like to visualize the structure of my git repository. It adds almost everything you need, you just need to explore it a bit.

Git client

These are really useful when you don’t understand it yet. Thier focus is to help you manage your git repositories, use branches, and many other things almost all visually. Helps reduce the need to understand and memorise yet another tool.

Of the Clients I currently recommend GitAhead. Followed by source-tree. There are many others, I suggest you try a few. I have had one or two git client apps really do a lot of damage but I was trying something stupid anyway.

Git Gui comes with git, not as intuitive but you dont have anything extra to install.

On the cmd line type git-gui or search with the search bar to find the app ~ you’ll have to select the new “repository” the first time, it will remember for the second time.

In the app select “Repository”, then select “visualize all branch history”
