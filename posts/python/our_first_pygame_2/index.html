Our first pygame - part 2

Date: 11.09.2023 - Author: Richard Porteous

Now we get to play a little. Lets simply draw a simple background and then download some premade shapes that we can move around.

First head over to kenney.nl and get the following images to follow along. The files we want are in the png/default folder. The larger images are 80x80. If you are adventurous try other images.

  • I’m going to use individual 2d png files for each item to start.
  • We load the image only once outside the game loop, so that helps.
  • we draw the image on the screen
  • we try move the image

Copy the blue_body_squircle.png from the default folder and place them in the a folder. if your images folder doesn’t exist, use VS Code to create the folder. I use an assets folder and create a player folder in it to store the player png files

Lets type the following code into our app above the first display update

#load images
player = pygame.image.load("assets/player/blue_body_squircle.png")
player_loc = player.get_rect()
player_loc.center = width/2, height/2

Then type this at the bottom of the game-loop , before the quitting of the window (NOTE must be indented the same as the rest of the game loop)

    #clear the display
    screen.fill(white)

    # place image on the screen
    screen.blit(player, player_loc)
    # apply changes
    pygame.display.update()

We can run this and should see our loaded image.

To move our image is simply a matter of checking for input events and then move the position of the game-object.

Inside the game loop just below the code checks for the quit event:

​        if event.type == KEYDOWN:
            # move user to the left
            if event.key in [K_a, K_LEFT]:
                player_loc = player_loc.move([-10 * dt, 0])
            # move user car to the right
            if event.key in [K_d, K_RIGHT]:
                player_loc = player_loc.move([10 * dt, 0])

As you see when running

  • it will leave images across the screen as you move left or right.
  • it requires to to press the key for each move

Not exactly what we want. lets fix the image being left behind problem.

After setting the window size write the following code

​black = (0,0,0)
white = (255,255,255)
screen.fill(white)
And ​​before the screen blit write this code.
    screen.fill(white)

If you had images in the background you could simply redraw them instead.

Run now and you’ll see the Game-Object move left or right each time you pressed the key.

Lets fix the need to press a key and change it so we can just hold the key down. Add this just before the loop

key_left = False
key_right = False

and replace the keydown event with this

Let’s be nice to other apps and limit our frame rate maximum to 60 frames per second.

# limit frame rate
# near our game_running variable we can add
clock = pygame.time.Clock()
FPS = 60


# and the top of our game loop we can add
​clock.tick(FPS)

So our complete listing now looks like this

import pygame
from pygame.locals import *

# set window size
size = width, height = (800, 600)
screen = pygame.display.set_mode(size)
black = (0,0,0)
white = (255,255,255)
screen.fill(white)


#load images
player = pygame.image.load("assets/player/blue_body_squircle.png")
player_loc = player.get_rect()
player_loc.center = width/2, height/2


# update the display to see what we set
pygame.display.update()


# Initialize the pygame code
pygame.init()

# Max frame rate
clock = pygame.time.Clock()
FPS = 60

leftkey = False
rightkey = False
upkey = False
downkey = False

speed = 10
# control variable
game_running = True


#game loop
while game_running:
    clock.tick(FPS)

    
    for event in pygame.event.get():
        if event.type == QUIT:
            game_running = False
            break
        
        if event.type==pygame.KEYDOWN:
            if event.key == K_ESCAPE:
                game_running = False
                break
            if event.key in [K_s, K_DOWN]:
                downkey = True
            if event.key in [K_w, K_UP]:
                upkey = True
            if event.key in [K_d, K_RIGHT]:
                rightkey = True
            if event.key in [K_a, K_LEFT]:
                leftkey = True
            
        elif event.type==pygame.KEYUP:
            if event.key in [K_s, K_DOWN]:
                downkey = False
            if event.key in [K_w, K_UP]:
                upkey = False
            if event.key in [K_d, K_RIGHT]:
                rightkey = False
            if event.key in [K_a, K_LEFT]:
                leftkey = False

    x=0
    y=0
    if leftkey:
        x += -speed
    if rightkey:
        x += speed
    if upkey:
        y += -speed
    if downkey:
        y += speed

    player_loc = player_loc.move([x, y])

    #clear the display
    screen.fill(white)

    # place image on the screen
    screen.blit(player, player_loc)
    # apply changes
    pygame.display.update()

# quit the pygame window
pygame.quit()
#the app exits
