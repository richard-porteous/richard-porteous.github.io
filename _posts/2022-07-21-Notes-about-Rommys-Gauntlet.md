## Notes on Rommys Gauntlet

OK, I admit it I wrote that clone of Chips Challenge. I don't plan re-writing at the moment. It has cost me a lot of time with very few rewards.


*HISTORY*
I wanted to write a game that my wife liked i.e. the MS clone of Chips Challenge with the same name.The old game wasn't avaiable then (Chips Challenge is now on steam by its creator).

I also had the desire to write games. I had just quit my job and was in a place where it just seemed difficult to get another job. The timing was perfect to experiment with games.

I've always liked coding. I like fixing old code - call me weird but there you go.

So I chose to code the game. I decided to use android and its SDK. So java was the natural choice. Google was on Android Gingerbread for the phones.

Just for fun (my kind of fun), I decided to grab bits of code from tutorials and slap them together mostly untouched to produce a game. It was a mess - man oh man. I picked up a book on refactoring and ended up with something half decent. With the help of a book by Mario Zechner and a small bit of my own code I produced 1.0 of Rommy's Gauntlet. I fixed bugs etc, but the game was jerky using canvas and too many npc's running around. Mario Zechner had made libGDX so I switched to using that library. It worked well enough, so I cleaned it up, changed the version to 1.1 and published so that the community and my wife could play the slightly improved game.

Unfotunately libGDX went through a period of not recieving updates and my app started to break on newer versions of Android. I dropped my plans to change to another engine and I discontinued the app recently.

I did try to reproduce it on GoDot and it was pretty good, but once again was jerky. I realized that a Game-engine is more than a library. I needed to write less code and use more of what was provided. 

So for a guy that likes to write software more than he likes to use built in concepts, this does present itself as a new chalenge.


### Quick Note on publishing.

First off this is very rewarding from the sense of acomplishment point-of-view. Google play store cost a one time 'deposit' of $25 which is refunded when you close the account. The last I checked Google-Development was still $25 after all these years. Apple works differently and its likely you will need an apple computer to compile and publish from ... though you will need to check.

It is funny how people like to be picky, rather than appreciate what you do. When I published the game, the creator (Chuck Sommerville) said he liked it when people copy his game. He was happy I was writing the clone. He also said not to use the name **Chip** anywhere as he did not have licence over that name anymore (you can read the full story ~ http://www.niffler.co.uk/Games/ChipsChallenge.htm plus on various fan sites).

I published the game ~ but felt bad about not mentioning the original game so I put the name on Google Play store. Guess what, Google doesn't like name dropping so they told me to take it and the authors name off.

A few weeks later I get this comment ~ I'm add-libbing because I don't feel like digging up the comment.
"You sucks", "You don't mention the original game anywhere" and on and on. 

So the lesson is, you can't please everyone, and for your sanity dont try to. The truth, if you need to credit anyone, do it in the actual game under Credits. Then on the Store mention that credits are in the game.
