## General info

### name conventions
This will help you stay sane :) and have others enjoy reading your code.

+ *camelCase* for variables: var camelCase
+ *SNAKE_UPPERCASE* for constants: const SNAKE_UPPERCASE
+ *camelCase* for functions: func camelCase: (lower *snake_case* is often used too)
+ *PascalCase* for classes: class_name MainPlayer
+ lower *snake_case* for actual file names
+ *PascalCase* for nodes: Node

Keep code as readable as possible, i.e. names must be self describing.
Don't be too clever with obscure code, unless you have no other choice - best place it in a self describing function and document well.

Some people like to show off with complicated bits of code and say it is the way you are supposed to program. I've been programming for 30 years and I couldn't disagree more. I've seen people get fired for messy and unreadable code. If you like that code use it sparingly, or better only when you want to show it to others but never in production code.
